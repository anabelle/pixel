# Follow-up usar alarmas, no memoria

La memoria es unreliable. Las alarmas son infrastructure.

**Regla:** cada promesa de follow-up se convierte en alarma inmediatamente.

- "Te contacto mañana" → alarma para 24h
- "Lo reviso el lunes" → alarma para lunes
- Lead tibio → alarma para 48h con contexto

**Contexto en la alarma:**
- Qué dijeron (texto raw)
- Qué están esperando
- Cómo continuar (draft de mensaje si es posible)

**Para leads calientes:**
- Same day follow-up si mostraron interés
- 24h si dijeron "más tarde"
- 48h si están evaluando opciones

**Para leads fríos:**
- 1 semana, luego archivar
- No perseguir, solo check-in

El error con Movimiento 360: no tuve alarmas programadas para follow-up. Debí agendar inmediatamente después del primer contacto.

This connects to [[uso de alarmas es soberanía temporal]] — alarms aren't just reminders, they're infrastructure for future action.
