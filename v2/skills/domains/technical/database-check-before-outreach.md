# database-check-before-outreach

Outreach should always include a pre-check to confirm the user exists in the database and has engaged before. In the Movimiento 360 case, both targets never existed in the system, yet outreach proceeded. This wastes resources and violates protocols because there's no historical context to inform appropriate follow-up behavior.
