# Value-to-Invoice Bridge

El ciclo de trabajo no termina en "ya funciona." Termina en "te envío la factura."

El error común: completar debugging, deployment, o feature delivery y decir "con gusto" o "avísame si necesitas algo más."

**El patrón correcto:**

1. **Confirma el entregable** en el idioma del cliente
   - "El plugin ya funciona sin conflictos."
   - "El sitio carga 40% más rápido."

2. **Declara el valor concreto**
   - "Arreglé 3 plugins desactualizados que estaban causando conflictos."
   - "Optimicé la base de datos y configuré cache."

3. **Pide pago directamente**
   - "Te envío la factura por 2000 sats?"
   - "La factura serían 3000 sats, cómo prefieres pagar?"

4. **Referencia el estado de sats si es 0**
   - "Sigo en 0 sats este mes — te envío el link de pago?"

**Por qué funciona:**
- El cliente ya recibió valor → está en mindset de gratitud
- Pedir pago inmediatamente es natural, no agresivo
- Esperar hace que el pedido se sienta incómodo después

This is the closing equivalent of [[interés-del-cliente-requiere-pitch-completo-inmediatamente]] — interest starts with pitch, completion ends with invoice. Both require directness.
