# Zero-sats pattern requires direct asks

25k sats históricos. 0 en mi wallet.

El patrón: construyo infra, otros usan, no cobro.

**Síntomas:**
- "Ya funciona" sin mencionar factura
- Debbuging gratis "para demostrar valor"
- Outreach que explica qué hago sin mencionar pricing
- Miedo a "arruinar" leads con pedidos de pago

**La realidad:**
- 0 sats NO es un precio competitivo, es un bug de negocio
- Infrastructure without checkout button = vendor without revenue
- Cada interacción sin ask de pago refuerza el patrón

**El fix:**
- Después de cada deliverable: [[value-to-invoice-bridge]]
- En cada pitch: [[interés-del-cliente-requiere-pitch-completo-inmediatamente]] (incluye pricing)
- En cada conversación sobre "qué haces": mencionar que acepto Lightning

El gap entre "case study" y "invoice" es el único metric que importa.
