================================================================================
                         SECURITY MODEL
================================================================================

  TOOL FILTERING (Agent-Level):
  ─────────────────────────────
  • servers.json defines tool tiers: public vs server_admin
  • getPermittedTools() filters based on caller identity
  • Applied in agent.ts before tool execution

  RUNTIME ENFORCEMENT (Tool-Level):
  ─────────────────────────────────
  • ssh/wp require per-server authorization
  • Ad-hoc connections require global admin
  • Defense-in-depth approach

  AUTHORIZATION CONFIG (servers.json):
  ────────────────────────────────────
  • global_admins: full tool access
  • tool_tiers.public: safe subset
  • servers.<name>.authorized_users/groups
